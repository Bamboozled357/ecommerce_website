Написать бэкенд для интернет-магазина. Проект должен называться ecommerce_website.
В проекте создать приложения shop и account
В приложении account создать следующие классы:
Profile с полями
user
is_sender (bool значение, пользователь продавец или нет)
В приложении shop создать следующие классы
Category с полями:
name
Item с полями
category
name
price
profile
Order с полями
item
quantity
profile
Необходимые возможности бэкенда:
По пути api/account/register/sender/ реализовать возможность зарегистрировать пользователя с необходимым профилем продавца
По пути api/account/register/buyer/ реализовать возможность зарегистрировать пользователя с необходимым профилем покупателя
По пути api/shop/category/ реализовать возможность просматривать все категории. Категории могут просматривать все, даже не авторизованные пользователи. А также по данному пути реализовать добавление Категории. Категорию могут добавлять только пользователи, у которых profile is_sender = True
По пути api/shop/category/<id>/ реализовать возможность просматривать детально категорию. Категорию могут просматривать все, даже не авторизованные пользователи. А также по данному пути реализовать изменение и удаление Категории. Категорию могут изменять и удалять только пользователи, у которых profile is_sender = True
По пути api/shop/category/<id>/item реализовать возможность просматривать все товары в данной Категории. Товары могут просматривать все, даже не авторизованные пользователи. А также по данному пути реализовать добавление Товаров. Товары могут добавлять только пользователи, у которых profile is_sender = True, при добавлении необходимый profile он должен брать от пользователя
По пути api/shop/category/<id>/item/<pk>/ реализовать возможность просматривать детально товар в данной Категории. Товар могут просматривать все, даже не авторизованные пользователи. А также по данному пути реализовать изменение и удаление Товара. Товары могут изменять и удалять только пользователи, которые сами создали этот товар.
По пути api/shop/catego ry/<id>/item/<pk>/order/ реализовать возможность заказать товар у данного продавца. Товар заказать могут только пользователи чей профиль is_sender = False.
По пути api/shop/category/<id>/item/<pk>/order/<order_id>/ реализовать изменение и удаление Заказа. Заказы могут изменять и удалять только пользователи, которые сами создали этот заказ.
Тип аутентификации сессионный и через токен
